# 🎬 LyricsSync

[한국어](README.md) · [English](README.en.md) · **日本語** · [中文](README.zh.md)

![version](https://img.shields.io/badge/version-v0.9.0--beta-orange) ![platform](https://img.shields.io/badge/platform-Windows-blue) ![license](https://img.shields.io/badge/license-MIT-green)

**AIミュージックビデオ制作のための、文字単位歌詞タイムコード抽出・編集ツール**

曲と歌詞を入れると波形を解析し、**歌詞の一文字一文字が正確に何秒に歌われるか**を
抽出。さらに波形上で直接ドラッグしながら補正できる、完全ローカルのWindows
デスクトップアプリです。

> **UI言語**: 한국어 / English / 日本語 / 中文 / Español / Français / Deutsch / Русский
> （ホーム画面左上で選択。使い方はアプリ内の `?` ボタン参照）。
> **楽曲の解析は言語を問いません** — 日本語・韓国語・英語・中国語・ロシア語など、
> 多言語MMS強制アライメント＋ローマ字化で処理します。
> CJKは文字単位、アルファベット言語は単語単位でタイミングが付きます。

---

## なぜ作ったか

AIでMVを作っていると、必ず同じ壁にぶつかります。

**「この歌詞、正確に何秒に歌われてる？」**

歌詞に合わせたタイポグラフィ、リップシンク、カット割り、モーション演出 — すべて
音節単位の正確なタイミングが必要です。しかしLLMに音源を入れてもこのタイミングは
取れません。毎回その部分の音声を切り出して解析を頼み、結果を手で直す繰り返しでした。

LyricsSyncはその繰り返しをなくします:

- 曲全体を**一度解析**すれば全文字のタイムコードが出て、
- 欲しい部分を**ドラッグすれば**その区間の正確な時間がすぐ出て、
- その値を **After Effectsのタイポグラフィ、Seedance動画生成、リップシンク**にそのまま使えます。

自動アライメントは100%完璧にはなり得ないため、このツールの本当の核は
**補正ワークフロー**です。大きなズレを数カ所直せば、残りは波形に合わせて
自動的に収束する設計になっています。

## パイプライン

```
音源 + 歌詞（歌詞は省略可 — AIが自動書き起こし）
   │
   ├─ ① ボーカル分離 ──── BS-RoFormer（SOTAステム分離）
   ├─ ② 歌詞自動抽出 ──── Whisper large-v3-turbo（歌詞未入力時）
   ├─ ③ 強制アライメント ─ MMS forced alignment（1,100+言語）
   │                        · 正解歌詞を波形にCTCで直接整列
   │                        · CJK＝文字単位、アルファベット＝単語単位
   │                        · ローマ字化（uroman）で言語を統一処理
   └─ ④ ビート解析 ────── Beat This!（BPM/拍/小節）＋キックオンセット
   │
   ▼
文字別タイムテーブル（project.json）
```

ASRで「書き起こす」のではなく、**既に分かっている正解歌詞を波形に強制整列**する
方式なので、音声認識ベースの字幕ツールより根本的に正確です。重いGPU計算
（音素確率マップ）は初回解析でキャッシュされ、以降の再整列はミリ秒で完了します。

## 機能詳細

### 自動解析
- **ボーカル分離**: BS-RoFormerでボーカルのみ抽出し整列精度を最大化。ドラムステムも分離（htdemucs）
- **歌詞自動抽出**: 歌詞欄を空にするとWhisperがボーカルから書き起こし（曲の言語指定可、既定は自動検出）
- **BPM/拍/キック**: Beat This!トランスフォーマー + librosaオンセット — 波形上にグリッド表示
- **タイムコードヒント**: 歌詞に `[Chorus @ 1:23]` や単独行 `@1:23` を書くと、初回解析が
  その付近（±3秒）を探索 — 概算でも初期精度が大幅アップ
- **Sunoタグ対応**: `[Intro]`, `[piano only]` などをそのまま貼り付ければセクション構造として解析

### 補正ワークフロー（核心）
- **カラオケハイライト**: 再生中の文字がリアルタイムに光る — ズレを耳と目で即発見
- **アンカー（`A`キー）**: 文字を選択し、聞こえた瞬間に `A` — その文字が固定（⚓）され、
  **前後の区間が波形に合わせて自動再整列**。大きいものから直せば小さいものが収束
- **波形チップ直接編集**（`가`ボタン）: 波形上に全文字が掴めるチップとして表示。
  どのチップでもクリック・ドラッグで即移動。重なると最大4段に積まれ常に掴める
- **ビーズ押し出しモデル**: 文字を押すと順序が崩れないよう隣が連鎖的に押される。
  開始点基準の衝突判定で、伸びた整列に振り回されない。アンカーは絶対に動かない壁
- **複数選択グループ移動**: チェックを入れてマーキードラッグで複数文字を束ね、
  間隔を保ったまま一括移動
- **区間再解析**: 波形をドラッグすると区間内の文字が緑色に表示され、ワンクリックで再解析。
  ボーカルステム → 原曲（セカンドオピニオン）→ 均等配置の3段戦略で、毎回結果レポート
  （文字数 · 信頼度 · 使用ソース）を表示
- **強制配置**: Shiftで歌詞範囲を選択してから波形区間をドラッグすると、
  その歌詞をその区間に強制整列 — アライナーが丸ごと見逃したパートの復旧用
- **右クリック再整列メニュー**: ここから前後を全体/N秒だけ再整列、文字削除
- **歌詞追加**: 文字の間にマウスを乗せると `+` ボタン — 挿入すると周辺が自動再整列
- **ダブルクリック精密編集**: 文字のテキストと開始/終了秒を小数点単位で直接入力
- **Ctrl+Z**: すべての編集（アンカー・移動・追加・削除・再解析）を取り消し

### 出力・連携
- **区間コピー**: クリック → Shift+クリックで範囲 → 絶対開始時間 + **相対タイムコード**
  （先頭文字 = 0.000）+ フレーム番号（fps設定可）をクリップボードへ。
  After Effectsのプリコンポワークフローに最適化
- **書き出し（プレビュー付き）**:
  - `TXT` — 人が読むタイムテーブル（セクション/行/文字別時間+フレーム）
  - `SRT` — 標準字幕。Premiere/AE/YouTube直行
  - `LRC` — 文字単位タグ付きカラオケ形式
  - `JSON` — 全生データ（LLM/スクリプト処理用）
- **LLM連携**: `projects/<名前>/project.json` をClaudeなどに直接読ませて
  AEタイポグラフィ自動化（.jsx生成）、リップシンク、カット割りリスト生成に活用

### アプリ
- コンソールウィンドウなしのWindowsデスクトップアプリ（ダークテーマ、DWMタイトルバー）
- 原曲 / ボーカルのみ / ドラムのみ を切り替えて再生
- ドラッグ＆ドロップで音源入力
- GPU自動検出（なければCPUフォールバック）— ホーム画面下部に状態表示
- プロジェクトの完全削除（残留ファイルなし）

## ショートカット

| キー | 動作 |
|---|---|
| `Space` | 再生/停止 |
| `A` | 選択文字を再生位置にアンカー |
| `Alt+波形クリック` | 選択文字をその位置にアンカー |
| `Shift+クリック` | 歌詞範囲選択（波形にも緑表示） |
| 波形ドラッグ | 区間再解析（ラベルOFF時） |
| チップドラッグ | 文字移動＋隣を押し出し（ラベルON時） |
| ダブルクリック | 精密編集（テキスト/時間） |
| 右クリック | 再整列メニュー |
| `Esc` | 選択/区間解除 |
| `Ctrl+Z` | 元に戻す |

## 動作環境

| 項目 | 推奨 | 最小 |
|---|---|---|
| GPU | NVIDIA（VRAM 6GB以上、CUDA 12.x） | なくても可 — CPU自動フォールバック |
| 解析速度（3〜4分の曲） | GPU: 約1〜2分 | CPU: 約10〜20分 |
| RAM | 16GB | 8GB |
| ディスク | モデルキャッシュ約4GB + 曲ごとにステム100〜200MB | |
| OS | Windows 11（WebView2内蔵） | Windows 10 + WebView2ランタイム |

- AMD/Intel GPUはCUDA非対応のためCPUモードで動作します。
- 初回起動時にAIモデル（BS-RoFormer、MMS、Whisper turbo、Beat This!）を自動ダウンロードします。

## インストール & 起動

```
git clone https://github.com/elecpure2/LyricsSync.git
cd LyricsSync
install.bat        ← ダブルクリック一回（venv + PyTorch + ffmpeg 自動インストール）
```

完了後、生成された `LyricsSync.lnk` をダブルクリック（コンソールなしで起動）。
Python 3.11+ が必要です。

## ワークフロー概要

1. **新規プロジェクト**: 名前 + 音源（D&D可）+ 歌詞（空欄=自動書き起こし）→ 解析開始
2. **聴く**: カラオケハイライトでズレを発見
3. **補正**: アンカー（A）→ チップドラッグ → 区間再解析 → 必要ならダブルクリック精密編集
4. **抽出**: 区間コピー（AE用相対タイムコード）または TXT/SRT/LRC/JSON 書き出し
