# 🎬 LyricsSync

**한국어** · [English](README.en.md) · [日本語](README.ja.md) · [中文](README.zh.md)

![version](https://img.shields.io/badge/version-v0.9.0--beta-orange) ![platform](https://img.shields.io/badge/platform-Windows-blue) ![license](https://img.shields.io/badge/license-MIT-green)

**AI 뮤직비디오 제작을 위한 글자 단위 가사 타임코드 추출·편집 툴**

곡과 가사를 넣으면 파형을 분석해서 **가사 한 글자 한 글자가 정확히 몇 초에 나오는지**를 뽑아주고,
파형 위에서 직접 드래그하며 교정할 수 있는 로컬 데스크톱 앱입니다.

> **Multilingual**: UI in 한국어 / English / 日本語 / 中文 / Español / Français / Deutsch / Русский
> — 홈 화면 좌측 상단에서 선택. 사용법은 앱 안의 `?` 버튼 참고.
> Song analysis is language-universal: Korean, English, Japanese, Chinese, Russian and most
> languages are aligned via multilingual MMS forced alignment + romanization.
> CJK lyrics are timed per character, alphabet languages per word.

## 왜 만들었나

AI로 뮤직비디오를 만들다 보면 반드시 부딪히는 문제가 있습니다.

**"이 가사가 정확히 몇 초에 나오지?"**

가사에 맞춘 타이포그래피, 립싱크, 장면 전환, 모션 이펙트 — 전부 음절 단위의 정확한
타이밍이 필요합니다. 그런데 LLM에 음원을 통째로 넣어도 이 타이밍은 제대로 잡아주지
못합니다. 그래서 매번 해당 구간의 음성을 잘라서 따로 분석을 맡기고, 그 결과를 다시
손으로 수정하는 과정을 반복해야 했습니다.

LyricsSync는 이 반복을 없애기 위해 만들어졌습니다:

- 곡 전체를 **한 번 분석**하면 모든 글자의 타임코드가 나오고,
- 원하는 부분을 **드래그로 집으면** 그 구간의 정확한 시간이 바로 나오고,
- 그 값을 **After Effects 타이포그래피, Seedance 영상 생성, 립싱크**에 그대로 사용합니다.

자동 정렬이 100% 완벽할 수는 없기 때문에, 이 툴의 진짜 핵심은 **교정 워크플로우**입니다.
큼직한 오류 몇 개만 사람이 잡아주면 나머지가 파형에 맞춰 자동으로 수렴하는 구조로
설계되어 있습니다.

## 핵심 파이프라인

```
음원 + 가사 (가사는 생략 가능 — AI가 자동 추출)
   │
   ├─ ① 보컬 분리 ──────── BS-RoFormer (SOTA 스템 분리)
   ├─ ② 가사 자동 추출 ─── Whisper large-v3-turbo (가사 미입력 시)
   ├─ ③ 강제 정렬 ──────── MMS forced alignment (1,100+ 언어)
   │                        · 정답 가사를 파형에 직접 CTC 정렬
   │                        · 한/일/중 = 글자 단위, 알파벳 언어 = 단어 단위
   │                        · 로마자화(uroman)로 언어 통합 처리
   └─ ④ 비트 분석 ──────── Beat This! (BPM/비트/마디) + 킥 온셋
   │
   ▼
글자별 타임테이블 (project.json)
```

ASR로 "받아쓰기"를 하는 것이 아니라 **이미 알고 있는 정답 가사를 파형에 강제
정렬(forced alignment)** 하는 방식이라, 일반적인 음성인식 기반 자막 툴보다 근본적으로
정확합니다. 무거운 GPU 연산(음소 확률맵)은 첫 분석 때 캐시되므로, 이후의 모든
재정렬·교정은 밀리초 단위로 끝납니다.

## 기능 상세

### 자동 분석
- **보컬 분리**: BS-RoFormer로 보컬만 추출해 정렬 정확도 극대화. 드럼 스템도 분리(htdemucs)
- **가사 자동 추출**: 가사를 비워두면 Whisper가 보컬에서 받아쓰기 (노래 언어 지정 가능, 기본 자동 감지)
- **BPM/비트/킥**: Beat This! 트랜스포머 + librosa 온셋 — 파형 위에 비트 그리드로 표시
- **타임코드 힌트**: 가사에 `[Chorus @ 1:23]` 또는 줄 사이 `@1:23`을 적으면 첫 분석이
  그 지점 근처(±3초)에서 해당 파트를 탐색 — 어림값이어도 초기 정확도가 크게 향상
- **Suno 태그 지원**: `[Intro]`, `[piano only]` 등 태그를 그대로 붙여넣으면 섹션 구조로 파싱

### 교정 워크플로우 (핵심)
- **노래방 하이라이트**: 재생 중인 글자가 실시간으로 빛나고 지나간 글자는 밝게 —
  어긋난 지점을 귀와 눈으로 즉시 발견
- **앵커 (A 키)**: 글자를 선택하고 들리는 순간 `A` — 그 글자가 고정(⚓)되고
  **앞뒤 구간이 파형에 맞춰 자동 재정렬**. 큰 것부터 잡으면 작은 것들이 수렴
- **파형 칩 직접 편집** (`가` 버튼): 파형 위에 모든 글자가 잡을 수 있는 칩으로 표시.
  아무 칩이나 클릭·드래그로 즉시 이동. 겹치면 최대 4단으로 쌓여 항상 집을 수 있음
- **구슬(bead) 밀림 모델**: 글자를 밀면 순서가 뒤집히지 않게 이웃들이 연쇄로 밀림.
  시작점 충돌 기준이라 정렬이 늘어진 글자에 휘둘리지 않고, 앵커는 절대 안 밀리는 벽
- **다중선택 그룹 이동**: 체크박스 켜고 마퀴 드래그로 여러 글자를 묶어
  간격을 유지한 채 통째로 이동
- **구간 재분석**: 파형을 드래그하면 구간 안 글자들이 초록으로 표시되고 원클릭 재분석.
  보컬 스템 → 원곡(2차 의견) → 균등 배치 순의 3단 전략으로, 매번 결과 리포트
  (글자 수 · 신뢰도 · 사용된 소스)를 표시
- **강제 배치**: Shift로 가사 범위를 선택한 뒤 파형 구간을 드래그하면
  그 가사들을 그 구간에 강제 정렬 — 정렬기가 통째로 놓친 파트 복구용
- **우클릭 재정렬 메뉴**: 여기서 뒤로/앞으로 전체 재정렬, 앞뒤 N초만 재정렬, 글자 삭제
- **가사 추가**: 글자 사이에 마우스를 올리면 `+` 버튼 — 빠진 가사(애드립 등)를 끼워 넣으면
  주변이 자동 재정렬
- **더블클릭 정밀 수정**: 칩/글자를 더블클릭하면 가사 텍스트 + 시작/끝 초를
  소수점 단위로 직접 입력
- **Ctrl+Z**: 모든 편집(앵커, 이동, 추가, 삭제, 재분석)을 되돌리기

### 추출 · 연동
- **구간 복사**: 글자 클릭 → Shift+클릭으로 범위 → 절대 시작시간 + **상대 타임코드**
  (첫 글자 = 0.000) + 프레임 번호(fps 설정 가능)가 클립보드로.
  프리컴프 단위로 작업하는 After Effects 워크플로우에 최적화
- **내보내기 (미리보기 포함)**:
  - `TXT` — 사람이 읽는 타임테이블 (섹션/줄/글자별 시간+프레임)
  - `SRT` — 표준 자막. 프리미어/애펙/유튜브 직행
  - `LRC` — 글자 단위 태그 포함 가라오케 포맷
  - `JSON` — 전체 원본 데이터 (LLM/스크립트 처리용)
- **LLM 연동**: `projects/<이름>/project.json`을 Claude 등에 직접 읽혀서
  애펙 타이포그래피 자동화(.jsx 생성), 립싱크 타이밍, 장면 컷 리스트 생성에 활용

### 앱
- 콘솔 창 없는 Windows 데스크톱 앱 (다크 테마, DWM 커스텀 타이틀바)
- 원곡 / 보컬만 / 드럼만 전환 재생
- 드래그 앤 드랍으로 음원 입력
- GPU 자동 감지 (없으면 CPU 폴백) — 홈 화면 하단에 상태 표시
- 프로젝트 완전 삭제 (잔여 파일 없음)

## 단축키

| 키 | 동작 |
|---|---|
| `Space` | 재생/정지 |
| `A` | 선택 글자를 현재 재생 위치에 앵커 |
| `Alt+파형 클릭` | 선택 글자를 클릭 위치에 앵커 |
| `Shift+클릭` | 가사 범위 선택 (파형에도 초록 표시) |
| 파형 드래그 | 구간 재분석 (`가` 꺼진 상태) |
| 칩 드래그 | 글자 이동 + 이웃 밀림 (`가` 켠 상태) |
| 더블클릭 | 가사/시간 정밀 수정 |
| 우클릭 | 재정렬 메뉴 |
| `Esc` | 선택/구간 해제 |
| `Ctrl+Z` | 되돌리기 |

## 요구 사양

| 항목 | 권장 | 최소 |
|---|---|---|
| GPU | NVIDIA (VRAM 6GB 이상, CUDA 12.x) | 없어도 됨 — CPU 자동 폴백 |
| 분석 속도 (3~4분 곡) | GPU: 약 1~2분 | CPU: 약 10~20분 |
| RAM | 16GB | 8GB |
| 디스크 | 모델 캐시 약 4GB + 곡당 스템 100~200MB | |
| OS | Windows 11 (WebView2 내장) | Windows 10 + WebView2 런타임 |

- AMD/Intel GPU는 CUDA가 없어 CPU 모드로 동작합니다.
- 첫 실행 시 AI 모델(BS-RoFormer, MMS, Whisper turbo, Beat This!)을 자동 다운로드합니다.

## 설치 & 실행

```
git clone https://github.com/elecpure2/LyricsSync.git
cd LyricsSync
install.bat        ← 더블클릭 한 번 (venv + PyTorch + ffmpeg 자동 설치)
```

설치가 끝나면 생성된 `LyricsSync.lnk` 더블클릭 (cmd 창 없이 실행됩니다).
Python 3.11+ 필요.

## 사용 흐름 요약

1. **새 프로젝트**: 이름 + 음원(드래그 앤 드랍 가능) + 가사(비우면 자동 추출) → 분석 시작
2. **듣기**: 노래방 하이라이트로 어긋난 지점 발견
3. **교정**: 앵커(A) → 칩 드래그 → 구간 재분석 → 필요하면 더블클릭 정밀 수정
4. **추출**: 구간 복사(애펙용 상대 타임코드) 또는 TXT/SRT/LRC/JSON 내보내기

## 프로젝트 구조

```
LyricsSync/
├─ main.pyw              # 데스크톱 런처 (pywebview, 서버는 별도 프로세스)
├─ app/
│  ├─ server.py          # 로컬 FastAPI
│  └─ pipeline/          # 분리·정렬·비트·파서·프로젝트 관리
├─ ui/                   # 에디터 프론트엔드 (i18n 8개 언어)
├─ bin/                  # 내장 ffmpeg
└─ projects/<이름>/       # 음원 사본 · 스템 · 정렬 캐시 · project.json
```

## 기술 스택

Python 3.11 · PyTorch (CUDA 12.8) · [BS-RoFormer](https://arxiv.org/abs/2309.02612) (보컬 분리) ·
[MMS forced alignment](https://ai.meta.com/blog/multilingual-model-speech-recognition/) (torchaudio) ·
[Whisper large-v3-turbo](https://github.com/openai/whisper) (가사 자동 추출) ·
[Beat This!](https://github.com/CPJKU/beat_this) (비트 트래킹) · uroman (로마자화) ·
FastAPI + pywebview + wavesurfer.js
