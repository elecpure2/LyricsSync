# 🎬 LyricsSync

[한국어](README.md) · [English](README.en.md) · [日本語](README.ja.md) · **中文**

![version](https://img.shields.io/badge/version-v0.9.0--beta-orange) ![platform](https://img.shields.io/badge/platform-Windows-blue) ![license](https://img.shields.io/badge/license-MIT-green)

**面向AI音乐视频制作的逐字歌词时间码提取·编辑工具**

放入歌曲和歌词，LyricsSync 会分析波形，告诉你**每一个字精确在几秒被唱出**，
并且可以直接在波形上拖动进行校正。完全本地运行的 Windows 桌面应用。

> **界面语言**: 한국어 / English / 日本語 / 中文 / Español / Français / Deutsch / Русский
> （主页左上角选择；应用内按 `?` 查看使用指南）。
> **歌曲分析不限语言** — 中文、韩语、英语、日语、俄语等均通过多语言 MMS 强制对齐
> + 罗马音化处理。中日韩歌词按字计时，字母语言按词计时。

---

## 为什么做这个

用AI制作MV时，总会撞上同一堵墙：

**"这句歌词到底是几秒唱的？"**

配合歌词的字体动画、对口型、镜头切换、动态特效——全都需要音节级的精确时间。
但把音频喂给LLM，它抓不准这个时间。以前只能每次把那段音频剪出来单独分析，
再手动修正结果，无限重复。

LyricsSync 消灭了这个循环：

- 整首歌**分析一次**，得到所有字的时间码；
- **拖选任意区间**，立刻得到该段的精确时间；
- 数值直接用于 **After Effects 字体动画、Seedance 视频生成、对口型**。

自动对齐不可能100%完美，所以这个工具真正的核心是**校正工作流**：
你只需修正几个大偏差，其余部分会自动向波形收敛。

## 流水线

```
音频 + 歌词（歌词可省略 — AI自动识别）
   │
   ├─ ① 人声分离 ──────── BS-RoFormer（SOTA音轨分离）
   ├─ ② 歌词自动提取 ──── Whisper large-v3-turbo（未提供歌词时）
   ├─ ③ 强制对齐 ──────── MMS forced alignment（1,100+语言）
   │                       · 将已知正确的歌词CTC对齐到波形
   │                       · 中日韩=按字，字母语言=按词
   │                       · 通过罗马音化（uroman）统一处理
   └─ ④ 节拍分析 ──────── Beat This!（BPM/节拍/小节）+ 底鼓onset
   │
   ▼
逐字时间表（project.json）
```

不是用ASR"听写"，而是**把你已知正确的歌词强制对齐到波形**——
比基于语音识别的字幕工具从根本上更准确。繁重的GPU计算（音素概率图）
在首次分析时缓存，之后的所有重对齐都在毫秒内完成。

## 功能详情

### 自动分析
- **人声分离**: BS-RoFormer 提取纯人声以最大化对齐精度；同时分离鼓轨（htdemucs）
- **歌词自动提取**: 歌词留空时由 Whisper 从人声识别（可指定歌曲语言，默认自动检测）
- **BPM/节拍/底鼓**: Beat This! transformer + librosa onset — 以网格显示在波形上
- **时间码提示**: 在歌词中写 `[Chorus @ 1:23]` 或单独一行 `@1:23`，首次分析会在
  该点附近（±3秒）搜索 — 粗略估计也能大幅提高初始精度
- **Suno标签支持**: 直接粘贴 `[Intro]`、`[piano only]` 等标签即解析为段落结构

### 校正工作流（核心）
- **卡拉OK高亮**: 播放时当前字实时发光 — 用耳朵和眼睛立刻发现偏差
- **锚点（`A`键）**: 选中一个字，在听到它的瞬间按 `A` — 该字被固定（⚓），
  **前后区间自动按波形重对齐**。修大的，小的自动收敛
- **波形芯片直接编辑**（`가`按钮）: 波形上所有字显示为可抓取的芯片，
  点击/拖动任意芯片即可移动。重叠时最多堆4层，始终可抓取
- **串珠推挤模型**: 拖动一个字时邻居连锁推开，顺序永不颠倒；
  以起始点判定碰撞，不受拉长的对齐span干扰；锚点是推不动的墙
- **多选整体移动**: 勾选多选后框选多个字，保持间距整体移动
- **区间重分析**: 在波形上拖选 — 区间内的字变绿显示，一键重分析。
  三级策略（人声轨 → 原曲二次意见 → 均匀分布），每次显示结果报告
  （字数 · 置信度 · 使用的来源）
- **强制放置**: 先Shift选择歌词范围，再拖选波形区间，即可将这些歌词
  强制对齐到该区间 — 用于修复对齐器整段漏掉的部分
- **右键重对齐菜单**: 从此向前/向后全部或仅N秒重对齐；删除字
- **添加歌词**: 鼠标悬停字间出现 `+` 按钮 — 插入后周边自动重对齐
- **双击精密编辑**: 直接输入字的文本和精确到小数的开始/结束秒
- **Ctrl+Z**: 撤销一切编辑（锚点、移动、增删、重分析）

### 节拍视图（歌词/节拍标签页）
- 编辑器下方在 **[歌词] / [节拍]** 标签间切换 — 节拍页将**强拍（小节）、节拍、底鼓**
  像歌词音节一样以芯片显示
- 点击定位播放，Shift+点击选范围，`复制区间`得到绝对开始＋相对时间码＋帧号
- 播放时当前节拍实时高亮 — 专为在字体动画之上叠加**节拍同步的闪烁/画面特效**设计

### 导出·联动
- **复制区间**: 点击 → Shift+点击选范围 → 绝对开始时间 + **相对时间码**
  （首字 = 0.000）+ 帧号（fps可设）复制到剪贴板。为 After Effects 预合成工作流优化
- **导出（带预览）**:
  - `TXT` — 人类可读的时间表（段落/行/逐字时间+帧）
  - `SRT` — 标准字幕，直入 Premiere/AE/YouTube
  - `LRC` — 带逐字标签的卡拉OK格式
  - `JSON` — 完整原始数据（供LLM/脚本处理）
- **LLM联动**: 把 `projects/<名称>/project.json` 直接交给 Claude 等，
  用于AE字体动画自动化（生成.jsx）、对口型时间、镜头切换清单

### 应用
- 无控制台窗口的 Windows 桌面应用（深色主题、DWM标题栏）
- 原曲 / 仅人声 / 仅鼓 切换播放
- 拖放输入音频
- GPU自动检测（无则CPU回退）— 主页底部显示状态
- 项目彻底删除（无残留文件）

## 快捷键

| 键 | 动作 |
|---|---|
| `Space` | 播放/暂停 |
| `A` | 将选中字锚定在播放位置 |
| `Alt+点击波形` | 将选中字锚定在该位置 |
| `Shift+点击` | 范围选择歌词（波形同步绿色显示） |
| 拖动波形 | 区间重分析（标签关闭时） |
| 拖动芯片 | 移动字+推挤邻居（标签开启时） |
| 双击 | 精密编辑（文本/时间） |
| 右键 | 重对齐菜单 |
| `Esc` | 取消选择/区间 |
| `Ctrl+Z` | 撤销 |

## 系统要求

| 项目 | 推荐 | 最低 |
|---|---|---|
| GPU | NVIDIA（显存6GB+，CUDA 12.x） | 可无 — 自动CPU回退 |
| 分析速度（3~4分钟歌曲） | GPU: 约1~2分钟 | CPU: 约10~20分钟 |
| 内存 | 16GB | 8GB |
| 磁盘 | 模型缓存约4GB + 每首歌音轨100~200MB | |
| 系统 | Windows 11（内置WebView2） | Windows 10 + WebView2运行时 |

- AMD/Intel GPU 无CUDA，以CPU模式运行。
- 首次运行自动下载AI模型（BS-RoFormer、MMS、Whisper turbo、Beat This!）。

## 安装 & 运行

```
git clone https://github.com/elecpure2/LyricsSync.git
cd LyricsSync
install.bat        ← 双击一次（自动安装 venv + PyTorch + ffmpeg）
```

完成后双击生成的 `LyricsSync.lnk`（无控制台窗口启动）。
需要 Python 3.11+。

## 工作流概要

1. **新建项目**: 名称 + 音频（可拖放）+ 歌词（留空=自动识别）→ 开始分析
2. **听**: 用卡拉OK高亮发现偏差
3. **校正**: 锚点（A）→ 拖芯片 → 区间重分析 → 需要时双击精密编辑
4. **提取**: 复制区间（AE用相对时间码）或导出 TXT/SRT/LRC/JSON

## 技术栈

Python 3.11 · PyTorch (CUDA 12.8) · [BS-RoFormer](https://arxiv.org/abs/2309.02612)（人声分离）·
[MMS forced alignment](https://ai.meta.com/blog/multilingual-model-speech-recognition/)（torchaudio）·
[Whisper large-v3-turbo](https://github.com/openai/whisper)（歌词自动提取）·
[Beat This!](https://github.com/CPJKU/beat_this)（节拍跟踪）· uroman（罗马音化）·
FastAPI + pywebview + wavesurfer.js
